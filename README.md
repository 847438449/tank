# tank

## 增量开发约束（音频链路）

> 以下约束来自当前需求，用于后续迭代时避免引入回归。

- 当前项目可运行基线：`690781c`
- 基线说明：`Fix loopback discontinuity via decoupled ring-buffer pipeline`
- 该基线被视为“最后一个可运行版本”

### 修改原则

1. **禁止重构/替换 audio capture 主链路**（仅允许局部、小步修改）
2. 所有改动必须以“**不破坏当前可运行状态**”为前提
3. 每次提交应尽可能小，并可快速回滚

### 优先级（严格按顺序）

1. 保持系统音频 loopback 可用
2. 在不影响 loopback 的前提下逐步添加：
   - 静音过滤
   - 时间戳保存
   - 防重复

### 建议流程（每次变更）

1. 先验证 loopback 基本可用
2. 仅引入一个最小功能点
3. 回归验证 loopback 与已有功能
4. 通过后再进入下一小步
